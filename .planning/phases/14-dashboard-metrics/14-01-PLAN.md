<task type="auto">
  <name>Task 2: Create dashboard query library</name>
  <files>src/lib/dashboard-queries.ts</files>
  <action>
    Create `src/lib/dashboard-queries.ts` as a server-only module (add `import "server-only"` at top if the project has that dependency, otherwise just keep it as a plain TS module — it will only be imported from server components).

    Implement these four exported async functions plus supporting types:

    **Type and helper:**
    ```typescript
    export type DateFilter = 'thisMonth' | 'thisQuarter' | 'last30' | 'allTime'

    export function getDateRange(filter: DateFilter): { from: Date; to: Date } | null {
      const now = new Date()
      switch (filter) {
        case 'thisMonth': return { from: startOfMonth(now), to: endOfMonth(now) }
        case 'thisQuarter': return { from: startOfQuarter(now), to: endOfQuarter(now) }
        case 'last30': return { from: subMonths(now, 1), to: now }
        case 'allTime': return null
      }
    }
    ```

    **getWinRateMetrics(filter: DateFilter = 'thisMonth'):**
    - Join deals → stages on stageId
    - Filter: isNull(deals.deletedAt) + stages.type IN ('won', 'lost')
    - Apply date range to deals.updatedAt (since no closedAt column exists — use updatedAt as proxy per RESEARCH.md)
    - Group by stages.type, count(deals.id)
    - Compute: won, lost, total, rate (null when total === 0, to show "no data")
    - Rate formula: `rate: total === 0 ? null : Math.round((won / total) * 100)`
    - Return: `{ won: number, lost: number, total: number, rate: number | null }`

    **getDealVelocityMetrics(filter: DateFilter = 'thisMonth'):**
    - Join deals → stages on stageId
    - Filter: isNull(deals.deletedAt) + eq(stages.type, 'won') + date range on deals.updatedAt
    - Select: `sql<number>\`AVG(EXTRACT(EPOCH FROM (${deals.updatedAt} - ${deals.createdAt})) / 86400.0)\`` aliased as avgDays
    - Also select count(deals.id) as count
    - Return: `{ avgDays: number | null, count: number }` (Math.round avgDays, null if no data)

    **getPipelineValueByStage():**
    - No date filter (shows current open pipeline snapshot, not historical)
    - Join deals → stages on stageId
    - Filter: isNull(deals.deletedAt) + eq(stages.type, 'open')
    - Group by stages.id, stages.name, stages.color, stages.position
    - Order by stages.position ASC
    - Select: stageName, stageColor, sum(deals.value) as totalValue, count(deals.id) as dealCount
    - IMPORTANT: sum() returns string | null — parse with parseFloat(r.totalValue ?? '0')
    - Compute total by summing all stage values
    - Return: `{ stages: Array<{ stage: string, color: string, value: number, count: number }>, total: number }`

    **getActivityCompletionMetrics(filter: DateFilter = 'thisMonth'):**
    - Three separate count queries run in parallel via Promise.all:
      1. completedCount: activities where isNull(deletedAt) + isNotNull(completedAt) + dueDate in range
      2. totalCount: activities where isNull(deletedAt) + dueDate in range
      3. overdueCount: activities where isNull(deletedAt) + isNull(completedAt) + lt(dueDate, new Date()) — NO date range filter on this one (all-time overdue)
    - Return: `{ completed: number, total: number, overdue: number, rate: number | null }`

    Import from drizzle-orm: count, sum, sql, eq, and, isNull, isNotNull, lt, gte, lte
    Import from date-fns: startOfMonth, endOfMonth, startOfQuarter, endOfQuarter, subMonths
    Import db from @/db
    Import deals, stages, activities from @/db/schema (use the correct schema import path — check how admin/page.tsx imports)

    Add a comment on the velocity function: "// Note: using updatedAt as proxy for close date — deals table has no closedAt column"
  </action>
