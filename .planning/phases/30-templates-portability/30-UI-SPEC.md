## Component Inventory

### 1. HTTP Template Selector (in HTTP config form)

**Location:** Top of `http-config.tsx`, above the Method field (D-01).

**Component:** shadcn `Select` with grouped items (`SelectGroup` + `SelectLabel`).

**Behavior:**
- Placeholder text: "Load from template..."
- Two groups: "Built-in" (static templates) and "Custom" (DB-stored templates)
- Selecting a template immediately overwrites all HTTP fields: method, url, headers, body, timeout, retryCount (D-02)
- No confirmation dialog on overwrite
- Selecting does NOT lock the fields -- user can still edit after template load
- "Custom" group only appears when custom templates exist
- Each item shows template name only; no description in the compact dropdown

**Visual:** Standard `Select` component, full width, matching existing form field width in side panel.

### 2. Save As Template Button + Dialog (in HTTP config form)

**Location:** Bottom of `http-config.tsx`, below the Retry Count field (D-05).

**Component:** shadcn `Button` (variant="outline", size="sm") triggering a shadcn `Dialog`.

**Button label:** "Save as Template"
**Button icon:** `Bookmark` (lucide) at 16px, left of label.

**Dialog contents:**
- Title: "Save as HTTP Template"
- Name field: `Input`, required, placeholder "e.g. Slack Webhook"
- Description field: `Textarea`, optional, placeholder "What this template does", max 2 rows
- Footer: "Discard" (variant="ghost") + "Save Template" (variant="default")
- On success: toast "Template saved", dialog closes, template selector refreshes
- On error: toast with error message, dialog stays open

### 3. Workflow Create Dialog (replaces direct create)

**Location:** Replaces current `NewWorkflowButton` inline behavior with a `Dialog` (D-10).

**Trigger:** Same "New Workflow" button with Plus icon on `/workflows` page.

**Dialog layout:**
- Title: "Create Workflow"
- Full-width "Blank Workflow" card at top: icon (FileText, 24px) + name + description, border style, hover highlight
- Separator with text: "Or start from a template"
- 2-column grid of template cards below (responsive: 1 column below 480px dialog width)
- Each template card: name (14px semibold), description (12px muted-foreground), fixed height of 80px
- Clicking a card creates the workflow from that template and navigates to editor
- Loading state: card shows spinner after click, other cards disabled

**Template cards (4 starter templates per D-11):**

| Template | Description |
|----------|-------------|
| Scheduled API Sync | Fetch data from an API on a schedule and update CRM records |
| Webhook Notifier | Receive a webhook and send a notification to your team |
| Data Pipeline | Transform incoming data and create CRM entities automatically |
| Email Digest | Send a periodic email summary of recent CRM activity |

### 4. Export Button (in editor toolbar)

**Location:** Right side of toolbar, before the Save button (D-14).

**Component:** shadcn `Button` (variant="outline", size="sm").
**Icon:** `Download` (lucide) at 16px, left of label.
**Label:** "Export JSON"

**Behavior:**
- Calls a client-side function that serializes the current workflow state from the editor store
- Includes: triggers, nodes (with edges derived from node ordering), metadata (name, description), schema version `pipelite/v1`
- Strips secrets from header values matching patterns: `Authorization`, `Bearer`, `Token`, `API-Key`, `Secret` (case-insensitive key match), replacing values with `{{PLACEHOLDER}}`
- Downloads a `.json` file named `{workflow-name}-export.json` (slugified)
- Shows toast: "Workflow exported as JSON"
- No server round-trip needed -- pure client-side serialization

### 5. Import Button (in editor toolbar)

**Location:** Right side of toolbar, next to Export button (D-14).

**Component:** shadcn `Button` (variant="outline", size="sm").
**Icon:** `Upload` (lucide) at 16px, left of label.
**Label:** "Import JSON"

**Behavior:**
- Opens a hidden `<input type="file" accept=".json">` via ref click
- Reads selected file as text, parses as JSON
- Validates: must have `schemaVersion: "pipelite/v1"`, must have `triggers` array and `nodes` array
- On valid: calls `createWorkflow` server action with imported data, name gets "(Imported)" suffix on conflict (D-17), navigates to new workflow editor
- On invalid: shows toast with appropriate error message (see copywriting)
- File picker allows single file only

### 6. Custom Template Management

**Location:** No dedicated management page. Templates are managed inline:
- **Create:** Via "Save as Template" in HTTP config form
- **Delete:** Via a small trash icon button next to custom template items in the Select dropdown (visible only to creator + admin per D-08)
- **Edit:** Not in v1 scope. Users delete and re-save.

**Delete flow:** Click trash icon -> `AlertDialog` confirmation -> server action delete -> toast -> dropdown refreshes.

## Interaction States

### HTTP Template Selector
| State | Visual |
|-------|--------|
| Default | Select with placeholder "Load from template..." |
| Open | Dropdown with grouped items, "Built-in" and "Custom" group labels in muted-foreground text-xs |
| Template selected | Select shows template name, all HTTP fields populated |
| No custom templates | "Custom" group hidden entirely |

### Save As Template Dialog
| State | Visual |
|-------|--------|
| Idle | "Save as Template" outline button at form bottom |
| Dialog open | Focus on name input, Save button disabled until name entered |
| Saving | Save button shows spinner, inputs disabled |
| Success | Dialog closes, toast appears |
| Error | Toast with error, dialog remains open |

### Workflow Create Dialog
| State | Visual |
|-------|--------|
| Idle | "New Workflow" button on workflows list page |
| Dialog open | Blank workflow card + 4 template cards visible |
| Card hover | Card border transitions to `--ring` color, subtle shadow |
| Card clicked | Clicked card shows spinner centered, all other cards get `opacity-50 pointer-events-none` |
| Creating | Full dialog has no close interaction until creation completes or fails |
| Error | Toast with error, dialog re-enables all cards |

### Export/Import Buttons
| State | Visual |
|-------|--------|
| Idle | Both buttons in outline style in toolbar |
| Exporting | Export button shows spinner for download duration (typically instant) |
| Importing (file selected) | Import button shows spinner until workflow created |
| Import error | Toast with error message, button returns to idle |
