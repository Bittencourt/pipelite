  <files>
    src/lib/execution/recursion.ts,

    src/lib/triggers/create-run.ts
  </files>
  <behavior>
    - getCurrentExecutionDepth() returns 0 when not inside a workflow execution
    - getCurrentExecutionDepth() returns the depth set by runWithExecutionDepth
    - runWithExecutionDepth(3, fn) makes getCurrentExecutionDepth() return 3 inside fn
    - Nested runWithExecutionDepth calls use the inner depth
    - createWorkflowRun with depth < 5 creates the run successfully with the depth value
    - createWorkflowRun with depth >= 5 creates a run with status "failed" and error "Recursion limit reached (5 levels)"
  </behavior>
  <action>
    1. Write tests FIRST in `recursion.test.ts` covering the behaviors above. Run -- must fail (RED).

    2. Create `src/lib/execution/recursion.ts`:
       - `const MAX_RECURSION_DEPTH = 5`
       - `const executionDepthStorage = new AsyncLocalStorage<number>()` (import from `node:async_hooks`)
       - `getCurrentExecutionDepth(): number` -- returns `executionDepthStorage.getStore() ?? 0`
       - `runWithExecutionDepth<T>(depth: number, fn: () => T | Promise<T>): T | Promise<T>` -- runs fn within AsyncLocalStorage context with given depth

    3. Update `src/lib/triggers/create-run.ts`:
       - Import `getCurrentExecutionDepth` and `MAX_RECURSION_DEPTH` from recursion.ts
       - Add optional `depth` parameter to `createWorkflowRun(workflowId, triggerEnvelope, depth?)`
       - If depth is not provided, read it from `getCurrentExecutionDepth()`
       - If effective depth >= MAX_RECURSION_DEPTH, insert a run with status "failed", error "Recursion limit reached (5 levels)", and completedAt = now. Return the failed run.
       - Otherwise, insert the run with the depth value in the new `depth` column

    4. Run tests -- must pass (GREEN).
  </action>
  <verify>

  </verify>
  <done>
    - AsyncLocalStorage tracks execution depth across async boundaries
    - getCurrentExecutionDepth returns 0 outside workflow execution context
    - createWorkflowRun enforces depth limit of 5 and creates failed runs at the limit
    - Depth value persisted in workflow_runs.depth column
  </done>
