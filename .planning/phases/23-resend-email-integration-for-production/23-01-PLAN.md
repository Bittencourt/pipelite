   - `userInvites` table with columns: `id` (text, PK, crypto.randomUUID default), `email` (text, notNull), `token` (text, notNull, unique), `invitedBy` (text, notNull, references users.id), `expiresAt` (timestamp mode date, notNull), `acceptedAt` (timestamp mode date, nullable), `createdAt` (timestamp, defaultNow, notNull).

   - `digestLog` table with columns: `id` (text, PK, crypto.randomUUID default), `sentAt` (timestamp, defaultNow, notNull), `userCount` (integer, notNull) -- tracks how many users received the digest, `weekStart` (timestamp mode date, notNull) -- the Monday start of the week this digest covers.
   - This table prevents duplicate weekly digest sends and survives container restarts (unlike a module-level variable).

6. Update `src/db/schema/_relations.ts` -- add relations for notificationPreferences (one-to-one with users), userInvites (many-to-one with users via invitedBy), and digestLog (standalone, no relations needed).
