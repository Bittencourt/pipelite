1. Add `inviteUser` server action to `src/app/admin/users/actions.ts`:
   - Accepts `email: string`. Validates with zod (valid email format).
   - Verify caller is admin (same pattern as approveUser).
   - Check if email already has an active user (users table, not deleted) -- return error if so.
   - Check if a pending invite already exists (not expired, not accepted) -- return error if so.
   - Generate token via `crypto.randomUUID()`.
   - Set expiresAt to 7 days from now.
   - Insert into userInvites table.
   - Fire-and-forget: `sendInviteEmail(email, token, session.user.name || 'Admin').catch(...)`.
   - Return `{ success: true }`.

2. Create `src/app/admin/users/invite-dialog.tsx`:
   - Client component with a dialog (shadcn Dialog).
   - Email input field, "Send Invite" button.
   - Calls inviteUser server action.
   - Shows success toast on success, error toast on failure.
   - Trigger button: "Invite User" with Mail icon (from lucide-react).

3. Update `src/app/admin/users/page.tsx`:
   - Add the InviteDialog component next to existing action buttons in the header area.

4. Update registration flow in `src/app/register/page.tsx`:
   - Read `invite` query parameter from URL.
   - If invite token is present, validate it: look up in userInvites table, check not expired, check not already accepted.
   - Pre-fill the email field from the invite record (read-only).
   - After successful registration: mark invite as accepted (`acceptedAt = new Date()`), and set user status to `approved` directly (skip pending_approval). Email verification is still required.
