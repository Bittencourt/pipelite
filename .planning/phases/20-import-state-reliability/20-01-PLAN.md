**Rewrite `src/lib/import/pipedrive-import-state.ts`:**

Remove the in-memory Map entirely. Keep the `ImportProgressState` interface unchanged (callers depend on it). Add an internal `ImportProgressData` interface for the JSONB shape. All functions become async.

Function-by-function rewrite:

- `createImportState(importId: string, userId: string)` -> `Promise<ImportProgressState>`:
  - **Add userId parameter** (callers will pass `session.user.id`)
  - Check for existing `running` session: `db.query.importSessions.findFirst({ where: eq(importSessions.status, 'running') })`. If found, throw `new Error("An import is already in progress")`
  - INSERT into import_sessions with id=importId, userId, status='idle', progress=default ImportProgressData, cancelled=false
  - Return reconstructed ImportProgressState

- `getImportState(importId: string)` -> `Promise<ImportProgressState | undefined>`:
  - SELECT from import_sessions by id
  - If not found, return undefined
  - Reconstruct full ImportProgressState from DB columns + JSONB progress (see research code example for shape reconstruction)
  - Calculate `currentProgress` from completedEntities/totalEntities

- `updateImportState(importId: string, updates: Partial<ImportProgressState>)` -> `Promise<void>`:
  - Read current row, merge updates into progress JSONB
  - Full row UPDATE with merged JSONB + any status/cancelled changes
  - Cap errors at 50 entries
  - Set `updatedAt: new Date()`

- `cancelImport(importId: string)` -> `Promise<void>`:
  - UPDATE import_sessions SET cancelled=true WHERE id=importId AND status='running'

- `isImportCancelled(importId: string)` -> `Promise<boolean>`:
  - SELECT cancelled FROM import_sessions WHERE id=importId
  - Return cancelled value, or false if not found

- `clearImportState(importId: string)` -> `Promise<void>`:
  - No-op (sessions kept for audit trail per user decision). Just update status if needed.

- `calculateProgress(state: ImportProgressState)` -> `number`:
  - Keep synchronous (pure computation, no DB needed)

- `incrementImportedCount(importId, entityType, count)` -> `Promise<void>`:
  - Read current session, modify progress.imported[entityType] += count, UPDATE row

- `addImportError(importId, entity, message, details?)` -> `Promise<void>`:
  - Read current session, check if errors.length < 50, push new error if under cap, UPDATE row
  - **details field:** The `ImportProgressData.errors` array stores `{ entity, message }` only (no details) to keep JSONB small. Drop the `details` parameter silently.

- `addReviewItem(importId, type, id, reason)` -> `Promise<void>`:
  - Read current session, push to progress.reviewItems, UPDATE row

Import `db` from `@/db`, `importSessions` from `@/db/schema`, `eq` from `drizzle-orm`.
