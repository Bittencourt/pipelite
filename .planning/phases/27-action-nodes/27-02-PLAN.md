    - Create deal: calls createDealMutation with interpolated field mapping, returns { output: { success: true, id, entity } }
    - Update person by ID: resolves targetId from {{trigger.data.personId}}, calls updatePersonMutation
    - Update by field lookup: lookupField="email" + lookupValue="{{trigger.data.email}}" queries DB to find ID, then calls updateMutation
    - Delete organization: calls deleteOrganizationMutation with resolved ID
    - Create activity: calls createActivityMutation with interpolated fields
    - Mutation failure: throws error with mutation error message (run should fail)
    - Unknown entity type: throws descriptive error
    - Unknown operation: throws descriptive error
    - Uses runWithExecutionDepth(currentDepth + 1, ...) around mutation calls
    - Uses workflow.createdBy from context._workflowUserId as the userId param
  </behavior>
  <action>
    1. Create `src/lib/execution/actions/crm.ts`:
       - Import all CRUD mutations from `@/lib/mutations`, `interpolate`/`interpolateDeep` from `./interpolate`, recursion helpers from `../recursion`, `registerAction` from `./index`, `db` from `@/db`
       - Build a mutation dispatch map: `{ deal: { create: createDealMutation, update: updateDealMutation, delete: deleteDealMutation }, person: {...}, organization: {...}, activity: {...} }`
       - Handler function:
         a. Extract `entity`, `operation`, `fieldMapping`, `targetId`, `lookupField`, `lookupValue` from config
         b. Interpolate fieldMapping values using interpolateDeep with context
         c. Get `userId` from `context._workflowUserId` (string -- set by engine from workflow.createdBy). If missing, throw "No userId in execution context"
         d. Get current depth via `getCurrentExecutionDepth()`
         e. For **create**: wrap mutation call in `runWithExecutionDepth(depth + 1, () => createMutation({ ...interpolatedFields, userId }))`. Return `{ output: result }` if success, throw `result.error` if not.
         f. For **update**: resolve target ID -- if `targetId` is set, interpolate it. If not, use `lookupField` + `lookupValue` to query the DB (e.g., `SELECT id FROM people WHERE email = ?`). Use Drizzle `db.select().from(table).where(eq(column, value)).limit(1)`. Then call updateMutation wrapped in runWithExecutionDepth.
         g. For **delete**: resolve target ID same as update. Call deleteMutation wrapped in runWithExecutionDepth.
       - For field lookup, build a simple entity-to-table+field resolver. Support common lookup fields: deal (title), person (email, name), organization (name), activity (subject). Use Drizzle `ilike` for text fields.
       - Register with `registerAction("crm_action", handler)`

    2. Update `src/lib/execution/actions/index.ts` to add `import "./crm"` to side-effect imports.

    3. Update engine.ts to set `context._workflowUserId = workflow.createdBy` when initializing context in `executeRun()` (after loading workflow, before walking nodes). This provides the userId for CRM mutations.
  </action>
