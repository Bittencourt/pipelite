## Integration Gaps

### 1. Webhook URL Display Bug (TRIG-05)

**File:** `src/app/workflows/[id]/edit/components/config-forms/trigger-config.tsx:202`
**Issue:** URL template is `${origin}/api/v1/webhooks/${trigger.secret}` but actual route is `POST /api/webhooks/in/${workflowId}/${trigger.secret}`
**Impact:** Users copy wrong URL, get 404. Webhook route itself works correctly.
**Fix:** One-line change — use `${origin}/api/webhooks/in/${workflowId}/${trigger.secret}` (workflowId already available on line 200)

### 2. toggleWorkflow Orphaned from UI (EXEC-01)

**File:** `src/app/workflows/[id]/edit/components/toolbar.tsx`
**Issue:** Editor toolbar calls `updateWorkflow` when toggling active state, which does a simple DB update. The `toggleWorkflow` server action (which also bulk-cancels waiting runs) is never called from any UI component.
**Impact:** Disabling a workflow from the editor leaves waiting (delayed) runs in "waiting" status indefinitely.
**Fix:** Toolbar should call `toggleWorkflow` when active transitions false, or `updateWorkflow` should replicate cancellation.
